## 📖 脚本说明

本脚本是一个 **用户脚本（UserScript）**，运行在 Tampermonkey（油猴）或 ScriptCat（脚本猫）等用户脚本管理器中。它会在你打开优学院课程页面时自动运行。

### 脚本原理

1. **注入**：脚本注入到优学院的课程学习页面中
2. **检测**：自动检测页面内的视频播放器和练习题
3. **执行**：控制视频播放、静音、调速；从本地题库匹配答案并填充
4. **翻页**：章节完成后自动点击「下一页」跳转
5. **防挂机**：定期模拟鼠标移动/键盘操作，防止被判定为挂机

### 需要提前了解

- 本脚本是 **纯前端脚本**，所有操作在浏览器本地完成
- 题库数据存储在 **浏览器本地**（localStorage + GM_setValue），**不会上传到任何服务器**
- 视频以 **1x 正常速度** 播放，不会触发平台的异常检测

## ✨ 功能详解

### 🎬 自动刷视频
- 自动检测页面上存在的视频播放器（`<video>` 标签）
- 自动点击播放按钮开始播放
- 自动将视频静音（可在 `CFG.AUTO_MUTE` 配置开关）
- 以 **1x 正常倍速** 播放，不加速不跳转，符合平台规则
- 每隔 30 秒输出一次播放进度
- 视频播放完毕后自动检测「已看完」状态并触发翻页

### 📝 自动答题
- 检测页面上的 `.question-wrapper` 元素识别题目
- 提取题目文本，去重、格式化后匹配本地题库
- 支持 **单选题 / 多选题 / 判断题** 三种题型
- 匹配成功则自动勾选答案并提交
- 匹配失败则跳过，最多尝试 3 次后自动翻页

### ⏭ 自动跳转
- 完成所有当前页面的视频和答题后，自动点击「下一页」
- 遇到弹窗（确定要切换页面、视频观看时长等）自动关闭
- 多个连续页面自动处理，无需人工干预

### 🧠 防检测机制
- 每 45 秒模拟一次鼠标移动（随机坐标）
- 偶尔触发鼠标滚轮滚动
- 偶尔模拟键盘按键（Shift 键）
- 所有模拟操作随机间隔，避免规律性被检测


### 常见使用场景

手动完成一些题目后，点击 **「📥 收集已答题目」**，脚本自动提取页面上的正确答案保存到本地。多点几次，题库就积累起来了。

## 🤖 全自动模式行为

开启全自动模式后，脚本的执行逻辑如下：

```
启动全自动
    │
    ├─ 检测是否有视频
    │   ├─ 有视频 → 自动播放（静音、1x倍速）
    │   │           └─ 等待播放完毕
    │   │              └─ 检测到「已看完」→ 自动翻页
    │   └─ 无视频 → 跳过
    │
    ├─ 检测是否有题目
    │   ├─ 有题目 → 匹配题库
    │   │           ├─ 匹配成功 → 自动勾选答案 → 提交
    │   │           └─ 匹配失败 → 跳过（最多3次后自动翻页）
    │   └─ 无题目 → 跳过
    │
    └─ 既无视频也无题目 → 自动翻页（2次检测后）
         └─ 跳转到下一节 → 重复以上流程
```

**特殊处理：**
- 遇到弹窗 → 自动点击「知道了」「确定」「确定离开」等
- 页面切换 → 重置题目去重集合，避免漏答
- 长时间无操作 → 模拟鼠标/键盘防挂机

## 📚 题库管理

### 数据存储位置

题库数据存储在：
- **浏览器 localStorage**（键名：`ul_qbank`）
- **ScriptCat/Tampermonkey 存储**（键名：`qbank`）

两者互为备份，确保数据不丢失。

### 题库格式

```json
[
  {
    "question": "一百年来,中国共产党团结带领中国人民进行的一切奋斗、一切牺牲、一切创造,归结起来就是一个主题( )。",
    "type": "single",
    "answer": "C"
  },
  {
    "question": "中国梦归根到底是( ),必须紧紧依靠人民来实现,必须不断为人民造福。",
    "type": "single",
    "answer": "B"
  }
]
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `question` | ✅ | 题目文本（会自动去除空白和格式差异进行匹配） |
| `type` | ✅ | 题型：`single`（单选）、`multiple`（多选）、`judge`（判断） |
| `answer` | ✅ | 答案：单选题填选项字母（A/B/C/D），多选题用 `|` 分隔（A|B|C），判断题填「正确」或「错误」 |

### 如何积累题库

**方法一：自动收集（推荐）**
1. 手动完成几道题
2. 点击控制面板的 **「📥 收集已答题目」**
3. 脚本会自动扫描页面上所有已完成题目，提取正确答案保存

**方法二：手动编写**
按 JSON 格式创建题库文件，通过「📂 导入题库」导入。

**方法三：共享题库**
- 其他人导出题库后，将 JSON 内容保存为 `.json` 文件
- 通过「📂 导入题库」加载
- 也可以通过「📤 导出题库」分享自己的积累

## ❓ 常见问题

### Q1：脚本没有反应怎么办？
1. 确认你已安装 Tampermonkey 或 ScriptCat
2. 确认脚本已启用（脚本管理器图标上应显示脚本数量）
3. 确认当前页面 URL 以 `ulearning.cn` 结尾
4. 刷新页面重试
5. 打开浏览器控制台（F12 → Console），查看是否有 `[助手]` 开头的日志

### Q2：视频不自动播放？
- 部分浏览器策略要求用户先与页面交互才能播放视频
- 脚本会自动尝试触发播放，如果仍不播放，请手动点击一次视频播放按钮
- 检查脚本面板顶部的视频状态显示

### Q3：题库不匹配题目？
- 题库匹配基于题目文本的精确比较（去除了空格和换行）
- 如果题目文字与题库不完全一致，可能匹配失败
- 尝试使用「📥 收集已答题目」从已完成页面收集答案

### Q4：如何删除脚本的本地数据？
- Tampermonkey：管理面板 → 找到脚本 → 存储选项卡 → 清空
- ScriptCat：管理面板 → 找到脚本 → 点击「存储」→ 清空
- 也可以直接清除浏览器 localStorage

### Q5：翻页时弹窗拦截？
脚本会自动处理以下弹窗：
- 「确定要切换页面」
- 「题目没有完成」
- 「视频观看时长」
- 其他「知道了」「确定」弹窗

### Q6：能否多开课程同时刷？
建议一次只开一个课程页面。多开可能导致登录状态冲突或 Cookie 失效。

## ⚠️ 注意事项

1. **学术诚信** — 本工具仅供学习参考，请确保你已经理解课程内容。使用本脚本产生的任何后果由使用者自行承担。

2. **正常倍速** — 视频以 **1x 正常速度** 播放，不会触发平台倍速检测。请勿自行修改脚本中的倍速参数。

3. **数据隐私** — 题库仅存储在 **浏览器本地**，脚本**不会**向任何服务器发送数据。你可以通过审查源代码验证这一点。

4. **Cookie 时效** — 平台登录 Cookie 有时效（通常为数天），如果脚本失效（如页面跳转到登录页），请重新登录。

5. **课程时效** — 已过期的课程（截止日期已过）可以浏览内容，但学习进度不会保存。

6. **兼容性** — 本脚本同时兼容 Tampermonkey 和 ScriptCat。
